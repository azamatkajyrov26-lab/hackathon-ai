# Схема субсидирования: AS-IS → TO-BE

## Процесс: Приобретение маточного поголовья КРС, овец, баранов-производителей

---

## Участники (swim lanes)

| Участник | Описание |
|----------|----------|
| **Услугополучатель (СХТП)** | Фермер / заявитель |
| **ИСС (subsidy.plem.kz)** | Информационная система субсидирования |
| **УСХ (ответственный специалист)** | Управление сельского хозяйства — проверка |
| **УСХ (Бухгалтер)** | Бухгалтерия УСХ — выплаты |
| **Казначейство (Подрядчик)** | Обработка платежей |

## AS-IS: Текущая схема

### ШАГ 1. Таймлайн приёма заявок
- Период приёма: **с 20 января по 20 декабря**
- Заявки принимаются только в рамках этого периода

### ШАГ 2. Регистрация / Авторизация
- **Проверка:** Имеется регистрация в ИСС (subsidy.plem.kz)?
  - **НЕТ** → Проходит регистрацию в ИСС → данные сохраняются в БД ИСС
  - **ДА** → Авторизуется в ИСС

### ШАГ 3. Открытие формы подачи
- Открывает форму подачи заявки на субсидию (данные из ИСС)

### ШАГ 4. Проверка статистических данных
- **Проверка:** Юр. лицо зарегистрировано в этом году?
  - **НЕТ** → Получает стат. данные по встречному обязательству из **ГИСС (gosagro.kz)** (интеграция)
    - **Проверка:** Данные имеются?
      - **НЕТ** → Уведомление: «Необходимо предоставить статистическую информацию» → Принимает к сведению (**стоп**)
      - **ДА** → Продолжает
  - **ДА** → Продолжает (новое юрлицо, нет прошлых данных)

### ШАГ 5. Выбор направления субсидирования
- Выбирает направление субсидирования и МИО (УСХ) — данные из ИСС

### ШАГ 6. Проверка регистрации в ИАС РСЖ
- Подтягиваются данные из **ИАС «РСЖ»**
- **Проверка:** Субъект-получатель зарегистрирован в ИАС РСЖ?
  - **НЕТ** → Уведомление: «Необходима регистрация в ИАС РСЖ» → Принимает к сведению (**стоп**)
  - **ДА** → Продолжает

### ШАГ 7. Формирование заявки
- Формирует заявку на субсидирование (данные сохраняются в ИСС)

### ШАГ 8. Контактные данные
- Указывает контактный номер телефона

### ШАГ 9. Проверка: является ли субъект СПК?
- **НЕТ (не СПК)** →
  - Получает учётные номера субъекта-получателя субсидий из **ЕАСУ** (интеграция)
  - **Проверка:** Есть ли учётные номера?
    - **НЕТ** → Уведомление: «Отсутствие учётного номера» → Принимает к сведению (**стоп**)
    - **ДА** → Выбирает учётный номер → Продолжает
- **ДА (СПК)** → Продолжает к шагу 10

### ШАГ 10. Получение данных о поголовье
- Получает данные о приобретённом поголовье из **ИАС «РСЖ»** (интеграция)
- Проверка на ранее полученные субсидии по данным животным

### ШАГ 11. Фильтрация данных по критериям
- **Овцы, бараны-производители:**
  - Возраст при покупке внутри страны: на момент даты продажи, указанный в племенном свидетельстве
  - Возраст при импорте: на момент постановки скота на карантинирование у продавца (экспортёра)
  - Бараны-производители: от 4 до 18 месяцев включительно
  - Маточное поголовье овец: от 4 до 18 месяцев включительно
  - Соотношение баранов к маточному поголовью:
    1. Вольная/ручная случка — не менее 20 маток и не более 30 маток на 1 барана
    2. Искусственное осеменение — 1 баран на 300 маток
    3. Докрытие при искусственном осеменении — 1 баран на 100 маток
- **Маточное поголовье КРС:**
  - При покупке внутри страны: на момент даты продажи, указанный в племенном свидетельстве
  - При импорте: на момент постановки крупного рогатого скота на карантинирование у продавца (экспортёра)
  - Тёлки: 6–18 месяцев включительно
  - Нетели: 13–26 месяцев включительно
- **Проверка:** Данные соответствуют критериям?
  - **НЕТ** → Уведомление: «Отсутствие поголовья для получения субсидий» → Принимает к сведению (**стоп**)
  - **ДА** → Продолжает

### ШАГ 12. Выбор поголовья для субсидирования
- Выбирает поголовье для субсидирования из отфильтрованного списка (данные из ИСС)

### ШАГ 13. Сверка данных поголовья (Шаг 2 на схеме)
- Получает и сверяет данные выбранного поголовья по количеству и параметрам
- Источники: **ИАС «РСЖ»** + **ИС «ИСЖ»**
- **Проверка:** Данные соответствуют?
  - **НЕТ** → Уведомление: «Несоответствие данных» → Принимает в работу → **Возврат к шагу 12**
  - **ДА** → Продолжает

### ШАГ 14. Указание стоимости (Шаг 3 на схеме)
- Указывает стоимость каждого приобретённого животного (данные сохраняются в ИСС)

### ШАГ 15. Расчёт суммы субсидии
- Высчитывает общую сумму причитающейся субсидии

### ШАГ 16. Проверка: Импорт из другой страны?
- **НЕТ** → Переход к выбору ЭСФ
- **ДА** → Переход к проверке земельных участков СПК

### ШАГ 17. Подтверждение ЭСФ (Шаг 4 на схеме)
- Получает все доступные ЭСФ на имя субъекта-получателя субсидии из **ИС «ЭСФ»** (интеграция)
- Выбирает соответствующие ЭСФ по договору
- **Проверка:** Общая сумма затрат по выбранным ЭСФ ≥ Общая сумма причитающейся субсидии?
  - **НЕТ** → Уведомление: «Недостаточная сумма ЭСФ для подтверждения субсидии» → Принимает в работу
  - **ДА** → Продолжает

### ШАГ 18. Сведения о земельных участках (Шаг 5 на схеме)
- **Проверка:** Используются ли земельные участки СПК?
  - **ДА** → Проверка: Члены СПК являются ЛПХ?
    - **ДА** → Вводит все ИИН/БИН членов СК → Получает сведения о ЗУ с/х назначения членов СПК из **ЕГКН**
      - Проверка: Зем. участки имеются?
        - **НЕТ** → Уведомление: «Отсутствие ЗУ» → Принимает к сведению (**стоп**)
        - **ДА** → Выбирает ЗУ из списка → Загружает документ подтверждения членства в СПК
    - **НЕТ** → Получает сведения о ЗУ с/х назначения из **ЕГКН**
      - Проверка: Зем. участки имеются?
        - **НЕТ** → Уведомление: «Отсутствие ЗУ» → Принимает к сведению (**стоп**)
        - **ДА** → Выбирает ЗУ из списка
  - **НЕТ** → Получает сведения о ЗУ с/х назначения из **ЕГКН**
    - (аналогичная проверка наличия ЗУ)

### ШАГ 19. Загрузка документов (Шаг 6 на схеме)
- **Проверка:** Импорт из другой страны?
  - **ДА** → Загружает:
    - Копия акта карантинирования у продавца (экспортёра)
    - Таможенная декларация на товары / заявление о ввозе товаров и уплате косвенных налогов и заявление на перевод средств
  - **НЕТ** → Загружает:
    - Копия договора купли-продажи
    - Копия документа, подтверждающего полную оплату стоимости

### ШАГ 20. Выбор счёта
- Выбирает номер счёта для получения средств

### ШАГ 21. Подтверждение данных
- Подтверждает достоверность данных и соглашается с требованиями (данные сохраняются в ИСС)

### ШАГ 22. Подписание ЭЦП
- Подписывает заявку с ЭЦП

### ШАГ 23. Обработка системой
- Система ИСС:
  - Присваивает ID номер заявки и сохраняет
  - Добавляет в лист ожидания

### ШАГ 24. Результат заявителю
- Вариант 1: Мотивированный отказ → Получает мотивированный отказ
- Вариант 2: Отправка письма на email о принятии заявки → Отправка результата в ЛК

### ШАГ 25. Частичная выплата и лист ожидания
- Уведомление с предложением о частичной выплате субсидии
- **Проверка:** Подтвердил частичную выплату?
  - **ДА** → В течение 2 рабочих дней производится частичная выплата. Остаток подлежит выплате при перераспределении или при поступлении средств в будущем → Уведомление об оплате → **Конец**
  - **НЕТ** → Заявка остаётся в листе ожидания → Отправка письма на email о переводе средств → Уведомление о переводе средств в ЛК

### ШАГ 26. Рассмотрение специалистом УСХ
- УСХ (ответственный специалист):
  1. Принимает заявку в работу
  2. Проверяет скан-копии документов
  3. Проверяет сведения по заявке (данные из ИСС)
  4. Подписывает результат ЭЦП (данные сохраняются в ИСС)
- **Проверка:** Всё верно?
  - **ДА** → Подписывает положительный результат ЭЦП → Продолжает к выплате
  - **НЕТ** → Возврат к подписанию результата (на доработку)

### ШАГ 27. Выплата через Казначейство
- **Проверка:** Достаточно средств для полной выплаты?
  - **ДА** → Формирует счёт на выплату и выгружает XML файл (в ИСС)
  - **НЕТ, но есть часть суммы** → Формирует счёт на частичную выплату (XML)
  - **НЕТ** → Ожидание доп. финансирования → Проверка: Средства поступили? → Цикл ожидания
- УСХ (Бухгалтер): Грузит XML файл в систему **ИС «Клиент-Казначейство»**
- Казначейство обрабатывает платёж, возвращает **Форму 5-15**
- УСХ получает информацию о выплаченных заявках
- Выставляет статус «Заявка оплачена» (в ИСС)
- Заявитель получает уведомление об оплате → **Конец процесса**

## TO-BE: Изменения с AI-скорингом

> Шаги 1–22 остаются **без изменений**. Ниже описаны только изменённые и новые шаги.

### NEW: ШАГ 22.1 — Автоматический AI-скоринг

Сразу после подписания ЭЦП система **автоматически** запускает скоринг:

#### 22.1.1. Hard Filters (жёсткие проверки)
15 автоматических бинарных проверок по 7 государственным системам:

| № | Проверка | Источник |
|---|----------|----------|
| 1 | Регистрация в ГИСС | ГИСС (gosagro.kz) |
| 2 | Наличие ЭЦП | ЕАСУ |
| 3 | Учётный номер в ИАС РСЖ | ИАС РСЖ |
| 4 | Земельный участок с/х назначения | ЕГКН |
| 5 | Регистрация в ИС ИСЖ (верифицированные животные) | ИС ИСЖ |
| 6 | Регистрация в ИБСПР | ИАС РСЖ |
| 7 | Подтверждение затрат через ЭСФ | ИС ЭСФ |
| 8 | Нет невыполненных обязательств | ГИСС |
| 9 | Нет блокировки | ГИСС + Applicant |
| 10 | Возраст животных в допустимом диапазоне | ИС ИСЖ |
| 11 | Животные не субсидировались ранее | ИС ИСЖ |
| 12 | Заявка подана в допустимый период | Приказ №108 |
| 13 | Сумма субсидии ≤ 50% стоимости | Приказ №108 |
| 14 | Минимальное поголовье соответствует | Приказ №108 |
| 15 | Нет дублирующих заявок в текущем году | SubsidyAI |

**Результат hard filters:**
- Если есть непройденные → **Уведомление заявителю** (тип: «информирование», НЕ отказ)
- По каждому непройденному фильтру — подробное описание: что проверялось, что найдено, что нужно сделать
- Заявка **продолжает рассматриваться** независимо от результата hard filters

#### 22.1.2. Soft Factors (мягкие факторы)
Расчёт 8 факторов с весами:

| № | Фактор | Источник | Вес |
|---|--------|----------|-----|
| 1 | Площадь с/х земли | ЕГКН | ×0.15 |
| 2 | Темп роста продукции | ГИСС | ×0.15 |
| 3 | Эффективность (сохранность поголовья) | ИС ИСЖ | ×0.15 |
| 4 | История субсидий | ГИСС | ×0.12 |
| 5 | ЭСФ подтверждение | ИС ЭСФ | ×0.10 |
| 6 | Валовая продукция (год ранее) | ГИСС | ×0.10 |
| 7 | Соответствие нормативу (Приказ №108) | Правила | ×0.10 |
| 8 | Финансовое состояние | Казначейство | ×0.13 |

Балл правил = Σ (Фактор_i × Вес_i), максимум 100

#### 22.1.3. AI-модель (Machine Learning)
- Алгоритм: XGBoost (градиентный бустинг)
- Обучена на исторических данных заявок
- Выдаёт: AI-балл (0–100) + уверенность (confidence)
- Объяснение: SHAP-значения для каждого фактора

#### 22.1.4. Гибридный итоговый балл
```
Итоговый балл = 0.6 × AI-балл + 0.4 × Балл правил
```

#### 22.1.5. Рекомендация AI

| Балл | Рекомендация | Действие |
|------|-------------|----------|
| ≥ 70 | ОДОБРИТЬ | Статус: approved (если бюджет есть) |
| 40–69 | НА РАССМОТРЕНИЕ | Статус: checking (комиссия решает) |
| < 40 | ОТКЛОНИТЬ | Статус: checking (комиссия решает) |

#### 22.1.6. Проверка бюджета
- Бюджет достаточен + рекомендация «ОДОБРИТЬ» → **approved**
- Бюджет исчерпан + рекомендация «ОДОБРИТЬ» → **лист ожидания**
- Остальные → **на рассмотрение комиссии**

### ШАГ 23 (ИЗМЕНЁН). Обработка системой
- Система ИСС:
  - Присваивает ID номер заявки и сохраняет
  - **Сохраняет результат скоринга:** балл, рекомендация, 8 факторов, SHAP-объяснение
  - Добавляет в лист ожидания **с рейтингом (сортировка по баллу скоринга)**

### ШАГ 24 (ИЗМЕНЁН). Результат заявителю
- **Экран успешной подачи** — номер заявки, статус, балл AI-скоринга, рекомендация
- Уведомление на email + в ЛК с подробностями
- Если есть замечания по hard filters → **подробное уведомление** с рекомендациями по каждому фильтру
- Ссылка на **страницу методологии скоринга** (формулы, объяснения, FAQ)
- Бейдж непрочитанных уведомлений в меню

### ШАГ 26 (ИЗМЕНЁН). Рассмотрение специалистом УСХ
- УСХ (ответственный специалист):
  1. Принимает заявку в работу
  2. **ВИДИТ полную карточку скоринга:**
     - Итоговый балл (визуальное кольцо: зелёный ≥70 / жёлтый 40-69 / красный <40)
     - Рекомендация AI (ОДОБРИТЬ / НА РАССМОТРЕНИЕ / ОТКЛОНИТЬ / ИНФОРМИРОВАНИЕ)
     - **SHAP-диаграмма** — waterfall: какие факторы повысили/понизили балл
     - **Детализация по 8 факторам** с прогресс-барами, весами, источниками
     - **Результаты 15 hard filters** — пройдено/не пройдено с деталями из систем
     - **Данные из 7 внешних систем** (ГИСС, ИАС РСЖ, ИС ИСЖ, ИС ЭСФ, ЕГКН, ЕАСУ, Казначейство)
     - **История заявок заявителя** — тренд, средний балл, успешность, хронология
     - **Информация о бюджете** — использовано, остаток, достаточность
  3. Проверяет скан-копии документов
  4. Проверяет сведения по заявке
  5. **Принимает решение с учётом скоринга** (одобрить / отклонить / доп. проверка / частично одобрить)
  6. Подписывает результат ЭЦП

### ШАГ 27 (ИЗМЕНЁН). Очередь выплат по рейтингу
- Заявки в листе ожидания **отсортированы по баллу скоринга** (высший балл — первый в очереди)
- При поступлении средств — выплата идёт **сверху вниз по рейтингу**
- Формирует счёт → XML → Казначейство → Форма 5-15 → Статус «Оплачена»

## Сводная таблица изменений AS-IS → TO-BE

| Аспект | AS-IS | TO-BE |
|--------|-------|-------|
| После подписания ЭЦП | Сразу в лист ожидания | AI-скоринг (автоматический) → балл → лист ожидания |
| Лист ожидания | Без приоритетов (FIFO) | Отсортирован по баллу скоринга (рейтинг) |
| Специалист УСХ | Видит только документы и данные заявки | Видит скоринг + SHAP + факторы + историю + бюджет |
| Заявитель после подачи | Получает отказ или принятие | Видит балл, рекомендацию, методологию, детальные уведомления по фильтрам |
| Hard filter fail | Автоматический отказ | Информирование (не отказ), заявка продолжает рассматриваться |
| Выплата при нехватке бюджета | По порядку поступления | По рейтингу скоринга (приоритет — высший балл) |
| Уведомления | Базовые (отказ/принятие) | Подробные с рекомендациями по каждому фильтру + бейдж в меню |
| Прозрачность | Заявитель не видит критерии | Страница методологии с формулами, FAQ, объяснениями |

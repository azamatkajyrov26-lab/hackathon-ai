# UI: Панель эмулятора

**URL:** `/emulator/`
**Доступ:** Админ

---

## Назначение

Панель управления эмулятором 7 внешних государственных систем. Позволяет:
- Просматривать сгенерированные данные
- Тестировать API эндпойнты
- Генерировать новые данные
- Просматривать статистику

## Секция 3: Карточка сущности (при клике "Открыть")

Показывает ВСЕ JSON-данные по всем 7 системам:

**Вкладки:**
1. ГИСС → JSON с валовой продукцией, обязательствами
2. ИАС РСЖ → JSON с регистрацией, историей субсидий
3. ЕАСУ → JSON с учётными номерами
4. ИС ИСЖ → JSON с животными (таблица)
5. ИС ЭСФ → JSON со счетами-фактурами
6. ЕГКН → JSON с земельными участками
7. Казначейство → JSON с платежами

Каждая вкладка показывает:
- Форматированный JSON (syntax highlighted)
- Кнопка "Копировать JSON"
- Кнопка "Редактировать" (inline edit)

## Секция 4: API Тестер

Интерактивная панель для тестирования каждого из 7 эндпойнтов:

### Выбор API

**Select:** ГИСС | ИАС РСЖ | ЕАСУ | ИС ИСЖ | ИС ЭСФ | ЕГКН | Казначейство

### Форма запроса

**Для выбранного API показываются нужные поля:**

#### ГИСС: check-obligations
| Поле | Тип | Пример |
|------|-----|--------|
| iin_bin | text | 123456789012 |
| year | number | 2025 |

#### ИС ИСЖ: verify-livestock
| Поле | Тип | Пример |
|------|-----|--------|
| iin_bin | text | 123456789012 |
| account_number | text | KZ-AGR-001234 |
| animals (JSON) | textarea | [{"tag_number": "KZ00123456", ...}] |

**Кнопка:** "Отправить запрос"

### Результат

```
POST /api/emulator/giss/check-obligations/
Status: 200 OK
Time: 45ms

Response:
{
  "registered": true,
  "gross_production_previous_year": 45000000,
  "gross_production_year_before": 42000000,
  "growth_rate": 7.14,
  "obligations_met": true,
  ...
}
```

## Секция 5: Управление данными

**Кнопки действий:**

| Действие | Описание | Команда |
|----------|----------|---------|
| Загрузить Excel | Импорт Excel-выгрузки в базу | `load_excel` |
| Генерировать данные | Создание синтетических данных для всех сущностей | `generate_data` |
| Создать тестовые заявки | Создание N заявок с автоскорингом | `create_test_applications` |
| Очистить данные | Удаление всех сгенерированных данных | `clear_emulated_data` |

Каждая кнопка запускает management command через HTMX, показывает прогресс и лог.

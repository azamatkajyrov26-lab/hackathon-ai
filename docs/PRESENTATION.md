# SubsidyAI — Интеллектуальная система субсидирования животноводства

## Проблема (15 баллов)

### Текущая ситуация
- **341 млрд тенге** — долг государства перед фермерами по субсидиям ([источник](https://kapital.kz/economic/145296/fermeram-po-meram-gospodderzhki-zadolzhali-341-mlrd-tenge.html))
- **38.7 млрд тенге** — задолженность только по животноводству
- **51%** долга сконцентрировано в 3 областях (Акмолинская, Костанайская, СКО)
- Фермеры ждут выплат **с 2023 года** ([источник](https://rus.azattyq-ruhy.kz/finansy/96675-zhdut-dengi-s-2023-goda-deputaty-vozmutilis-dolgami-po-subsidiiam-v-340-mlrd-tenge))

### Корневая причина: FIFO (первый пришёл — первый получил)
По Приказу №108 МСХ РК выплата субсидий осуществляется **в порядке очерёдности по дате подачи**. Это означает:
- Неэффективные хозяйства получают субсидии наравне с эффективными
- Нет приоритизации по вкладу в экономику
- Ограниченный бюджет расходуется неоптимально
- 6 точек в процессе работают по FIFO — ни одна не учитывает данные

### Что решает SubsidyAI
Заменяет **FIFO на merit-based приоритизацию** во всех 6 точках процесса на основе данных из **7 государственных информационных систем**.

## Работа с данными (15 баллов)

### 7 государственных систем — источники данных
| # | Система | Что берём | Для чего |
|---|---------|-----------|----------|
| 1 | **ГИСС** | Встречные обязательства, рост продукции | Проверка добросовестности |
| 2 | **ИАС РСЖ** | История субсидий, регистрация | Надёжность фермера |
| 3 | **ЕАСУ** | Учётные номера, ЭЦП | Верификация личности |
| 4 | **ИС ИСЖ** | Поголовье, бирки, возраст животных | Масштаб хозяйства |
| 5 | **ИС ЭСФ** | Электронные счета-фактуры | Финансовая прозрачность |
| 6 | **ЕГКН** | Земельный кадастр, площадь | Земельные ресурсы |
| 7 | **Казначейство** | Платежи, налоги | Финансовая эффективность |

### Обработка данных
- **500 эмулированных хозяйств** с реалистичными данными из всех 7 систем
- **300 тестовых заявок** с полным набором признаков
- **18 hard-фильтров** (бинарные проверки: зарегистрирован/нет, заблокирован/нет)
- **8 soft-факторов** (непрерывные оценки 0-100 по каждому аспекту)
- **SHAP-объяснения** для каждого признака ML-модели

### Признаки ML-модели
- История субсидий (кол-во, успешность, суммы)
- Размер хозяйства (поголовье, площадь, нагрузка на пастбища)
- Финансовые показатели (обороты ЭСФ, налоги)
- Эффективность (продуктивность, падёж скота)
- Регистрационные данные (ГИСС, ИАС РСЖ, ЕАСУ)
- Земельные ресурсы (площадь с/х земли, кадастр)
- Идентификация животных (% с бирками)

## Модель & логика (20 баллов)

### Гибридный AI-скоринг
```
Итоговый балл = ML модель × 0.6 + Правила (8 факторов) × 0.4
```

**ML-модель (60%):**
- GradientBoosting (scikit-learn)
- Обучена на исторических данных субсидий
- SHAP-интерпретация каждого решения
- Уверенность модели отображается

**Правила (40%) — 8 факторов:**
1. История субсидий — доля успешных заявок (ИАС РСЖ)
2. Контробязательства — выполнение встречных обязательств (ГИСС)
3. Размер хозяйства — поголовье + нагрузка на пастбища (ИС ИСЖ, ЕГКН)
4. Эффективность — продуктивность + падёж (ГИСС, ИС ИСЖ)
5. Финансовая устойчивость — обороты, налоги (ЭСФ, Казначейство)
6. Регистрация — наличие в госреестре (ЕАСУ)
7. Земельные ресурсы — наличие земли (ЕГКН)
8. Идентификация животных — % идентифицированных (ИС ИСЖ)

### Merit Score для выплат
```
Merit Score = 30% × масштаб хозяйства
            + 25% × финансовая эффективность
            + 25% × история субсидий
            + 20% × приоритет направления (политика МСХ)
```

### 6 точек FIFO → Merit-based
| # | Точка процесса | Было (Приказ №108) | Стало (SubsidyAI) |
|---|----------------|-------------------|-------------------|
| 1 | Очередь проверки специалистом | По дате подачи | По AI-скору |
| 2 | Рассмотрение комиссией | По дате подачи | По AI-скорингу |
| 3 | Очередь на выплату | По дате одобрения | По Merit Score |
| 4 | Нехватка бюджета (перенос) | По дате на след. месяц | По Merit Score |
| 5 | Частичная выплата | Предложение по дате | По Merit + UI |
| 6 | Перераспределение бюджета | Вручную между регионами | Данные дефицит/профицит |

### 18 hard-фильтров (автоматические проверки)
- Фильтры 1-11: Базовые (регистрация ГИСС, ЭЦП, ИАС РСЖ, земля, животные, ЭСФ, обязательства, блокировка, возраст, дубли)
- Фильтры 12-15: Нормативные (период подачи, сумма, минимум поголовья, дубликаты)
- Фильтры 16-18: Регуляторные (падёж по Приказу №3-3/1061, нагрузка по Приказу №3-3/332, племенное свидетельство по Приказу №108)

## Explainability (15 баллов)

### Для заявителя (фермера)
- **Статус-баннер**: визуальный flow Подана → Специалист → Комиссия → Решение
- **AI рекомендация**: чётко написано "РЕКОМЕНДАЦИЯ" (не окончательное решение)
- **Балл**: кольцо с числом из 100, цветовая кодировка (зелёный/жёлтый/красный)
- **18 проверок**: каждая с объяснением что проверяется, результат, что делать если не пройдено
- **8 факторов**: раскрываемые карточки с формулами расчёта и исходными данными
- **SHAP waterfall**: визуализация вклада каждого параметра в оценку AI
- **Формула**: ML × 0.6 + Правила × 0.4 = Итого, с числами
- **Порог**: 70+ одобрить, 40-69 на рассмотрение, <40 отклонить

### Для специалиста МИО
- **Чек-лист**: 6 пунктов проверки (данные, документы, суммы, скоринг, земля, животные)
- **Сводка**: ключевые данные заявителя, субсидии, AI-скоринг в 3 карточках
- **Hard-фильтры**: быстрый статус — все пройдены или есть замечания
- **Кнопка**: "Проверено — передать комиссии" (активна только после всех галочек)

### Для комиссии
- **Ранжирование** по AI-скору, а не по дате
- **Решение**: одобрить / отклонить / доп. проверка / частично одобрить + обоснование
- **Бюджет**: доступный бюджет по региону и направлению

### Для казначейства (очередь выплат)
- **Merit Score**: визуальная разбивка по 4 компонентам с мини-барами
- **Граница бюджета**: красная линия отсечения в таблице
- **Региональная разбивка**: дефицит/профицит каждого региона
- **Частичная выплата**: кнопка с расчётом доступной суммы

## Technical Implementation (15 баллов)

### Стек
- **Backend**: Django 5.1 + Python 3.12
- **Database**: PostgreSQL 16
- **Cache**: Redis 7
- **ML**: scikit-learn (GradientBoosting) + SHAP
- **Frontend**: Alpine.js + Tailwind CSS
- **Deploy**: Docker Compose + Gunicorn
- **Server**: Ubuntu на 109.235.119.92

### Архитектура
```
Заявитель → Форма заявки → AI Скоринг (18 фильтров + 8 факторов + ML)
         → Специалист МИО (чек-лист, merit-based очередь)
         → Комиссия (ранжирование по скору)
         → Платёжный flow (Merit Score очередь)
         → Казначейство → Выплата
```

### Ключевые файлы
- `apps/scoring/scoring_engine.py` — AI-скоринг + Merit Score
- `apps/scoring/hard_filters.py` — 18 hard-фильтров
- `apps/scoring/soft_factors.py` — 8 soft-факторов
- `apps/scoring/ml_model.py` — ML модель + SHAP
- `apps/scoring/views.py` — все view-функции
- `apps/emulator/` — эмулятор 7 гос. систем

### Стабильность
- Автоматический entrypoint: миграции → seed_data → генерация 500 entities → 300 заявок → ML обучение → сервер
- Демо-аккаунты всегда работают (пароли сбрасываются при каждом запуске)
- Docker Compose: одна команда для запуска всего

### Зависимости
- Docker + Docker Compose
- Python 3.12 (внутри контейнера)
- PostgreSQL 16, Redis 7
- scikit-learn, shap, pandas, numpy

### Данные
- **500 эмулированных хозяйств** — синтетические данные, имитирующие реальные из 7 гос. систем
- **300 тестовых заявок** — автоматически сгенерированы с разными сценариями
- **Модель обучается** при каждом запуске на этих данных
- Данные НЕ реальные — это демо/эмуляция для хакатона

### Ограничения
1. **Эмуляция, не интеграция** — данные из 7 систем эмулируются, реальных API нет
2. **ML модель** обучена на синтетических данных — в production нужны реальные
3. **Merit Score** — инновационное предложение, требует изменения Приказа №108 для внедрения
4. **Частичная выплата** — эмуляция, реальная интеграция с Казначейством не реализована
5. **Один регион** — демо показывает Астану, масштабирование на все 17 регионов потребует доработки

### Сценарии применения
1. **МИО (Местный исполнительный орган)** — приём и обработка заявок на субсидии
2. **Комиссия по субсидированию** — ранжированное рассмотрение заявок
3. **Казначейство** — приоритизированная очередь выплат при ограниченном бюджете
4. **Фермеры** — прозрачная подача заявок с мгновенным AI-скорингом и объяснением
5. **МСХ РК** — аналитика распределения субсидий по регионам и направлениям

### Нормативная база
- **Приказ №108 МСХ РК** от 15.03.2019 — Правила субсидирования животноводства
- **Приказ №3-3/332** — Нагрузка на пастбища
- **Приказ №3-3/1061** — Нормы падежа скота
- Изменения 2025: Приказ №428 от 18.11.2025 — увеличение норм на 75%

## Итого: что даёт SubsidyAI

| Метрика | Было (FIFO) | Стало (SubsidyAI) |
|---------|------------|-------------------|
| Приоритизация | По дате подачи | По эффективности (Merit Score) |
| Прозрачность | Чёрный ящик | AI объясняет каждое решение (SHAP) |
| Скорость проверки | Ручная, дни | AI-скоринг мгновенно |
| Дефицит бюджета | Перенос по дате | Частичная выплата + перераспределение |
| Данные | 1 система (ГИСС) | 7 систем интегрированы |
| Hard-фильтры | Ручные проверки | 18 автоматических |
| Очередь выплат | FIFO | Merit Score (масштаб + финансы + история + направление) |

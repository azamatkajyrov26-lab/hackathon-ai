# SubsidyAI — Полное исследование, интеграции и план развития

## 1. ИДЕЯ ПРОЕКТА

SubsidyAI — интеллектуальная система скоринга сельскохозяйственных субсидий РК.
Ключевая инновация: не просто цифровизация подачи заявок, а **AI-скоринг + RFID мониторинг**,
что превращает процесс субсидирования из ручного в автоматический и прозрачный.

### Цитата inDrive:
> "Для inDrive уникальность и инновационность важнее технической сложности.
> Не стоит гнаться за сложным кодом – куда важнее найти новый вектор развития."

### Наш новый вектор:
- RFID-станции на фермах → автоматический мониторинг сохранности поголовья
- Real-time данные → сигнал в скоринг (дополнительный балл)
- Замена ручных ежеквартальных инспекций комиссией из 3+ человек на автоматику

## 2. ОБЯЗАТЕЛЬСТВА ФЕРМЕРА ПРИ ПОДАЧЕ НА СУБСИДИЮ

По Приказу №108 МСХ РК (+ поправки июнь 2025):

### Сохранность поголовья:
| Что получил | Срок сохранности |
|-------------|-----------------|
| Племенное маточное поголовье (в ИСЖ и ИБСПР) | 2 года с даты подачи |
| Племенные производители (в ИСЖ и ИБСПР) | 18 месяцев с даты подачи |
| Племенное стадо птицы (в ИБСПР) | 60 недель с даты подачи |

### Обязательства по продукции:
- Рост или сохранение объёма валовой с/х продукции
- Формула: Объём произведённой продукции × Цена
- Фиксируется в ГИСС

### Санкции:
- 1-й раз (2 года подряд невыполнение): бан 1 год
- Повторно: бан 2 года
- Несохранение скота → частичный возврат субсидии
- Долги автоматически переносятся в бюджет следующего года

## 3. ПОЛНАЯ КАРТА ИНТЕГРАЦИЙ — РЕАЛЬНЫЕ ДАННЫЕ

### 3.1. ГИСС (gosagro.kz) — Государственная информационная система субсидирования

**Оператор:** Министерство сельского хозяйства РК
**Назначение:** Единая платформа для ВСЕХ сельскохозяйственных субсидий. Заменила платную Qoldau с 01.01.2023. Бесплатна.

| Поле в нашей эмуляции | Тип | Реально | Описание |
|---|---|---|---|
| `registered` | bool | ДА | Регистрация на gosagro.kz обязательна |
| `blocked` | bool | ДА | Блокировка за невыполнение обязательств |
| `block_reason` | string | ДА | Причина блокировки |
| `obligations_met` | bool | ДА | Уровень выполнения фиксируется в ГИСС |
| `obligations_required` | bool | ДА | Применяется к получателям >100М тг |
| `total_subsidies_received` | number | ДА | Полный учёт всех выплат |
| `gross_production_previous_year` | number | ДА | Отчёт до 20 января |
| `gross_production_year_before` | number | ДА | Для расчёта динамики |
| `growth_rate` | number | ДА | Год к году сравнение |
| `consecutive_decline_years` | number | ДА | Определяет срок бана |
| `repeat_violation` | bool | ДА | Повторное нарушение → 2 года бана |

**Автоматические проверки ГИСС:**
- Расчёт суммы субсидии по нормативам (без человеческого фактора)
- Проверка бюджета региона в реальном времени
- Очередь заявок (порядковый номер)

**Интегрированные базы в реальном ГИСС:**
- МСХ — земельные участки, данные о скоте, техника
- МинФин — ИС ЭСФ, данные о кооперативах
- МинЮст — реестр юрлиц
- МЦРИАП — цифровые данные
- МВД — проверка личности
- Единый кадастр — земля
- ГБД "Юрлица" — членство в СПК
- Карта полей — спутниковая верификация посевов

## 5. ПРОЦЕСС ПОДАЧИ ЗАЯВКИ (AS-IS по Приказу №108)

### Шаг 1: Предрегистрация
- Регистрация в ИАС (plem.kz) с ЭЦП
- Загрузка свидетельства о регистрации
- Выбор категорий скота
- Заполнение модуля картографии животных

### Шаг 2: Регистрация на портале субсидий
- subsidy.plem.kz (животноводство) или gosagro.kz (остальное)
- Вход через ЭЦП
- Проверка и предоставление доступа в течение 1 часа

### Шаг 3: Заполнение заявки
- Данные организации (наименование, ИИН/БИН, адрес, руководитель, форма)
- Статистические данные (поголовье за прошлый год, выпуск продукции, производительность)
- Банковские реквизиты
- Проверка бюджета текущего года по региону

### Шаг 4: Подача заявки
- Через "Подать заявку" в региональное управление с/х
- Подписание ЭЦП

### Шаг 5: Автоматическая верификация
- Синхронизация с ИСЖ (ветеринарный реестр)
- Сверка с ИБСПР (племенная база)
- Сверка с ИС ЭСФ (электронные счёт-фактуры)
- ГИСС проверяет землю через кадастр
- Проверка членства в СПК через ГБД "Юрлица"
- Проверка ранее субсидированных животных
- Расхождения данных (ID, пол, порода, дата рождения) между ИБСПР и ИСЖ = ОТКАЗ

### Шаг 6: Решение
- Обработка в течение 2 рабочих дней (если есть бюджет)
- Перечисление на банковские реквизиты
- Нет бюджета → заявка в очередь ожидания
- Заявитель может отозвать, исправить и перенаправить до одобрения

### Шаг 7: Постмониторинг
- Ежеквартальный мониторинг сохранности поголовья
- Отчёты до 10-го числа следующего месяца после квартала
- Выездные проверки крупных объектов комиссией из 3+ специалистов
- Отчёт по валовой продукции до 20 января и по итогам года

## 6. RFID-ИННОВАЦИЯ — ПЛАН ВНЕДРЕНИЯ

### 6.1. Текущая ситуация
- КРС, овцы, козы, верблюды, свиньи → ушные бирки (ручные)
- Лошади → таврение или чипирование (уже электронное)
- Мониторинг → ручной, ежеквартально, комиссия 3+ чел.

### 6.2. Наше предложение
RFID-станции на фермах (проходные ворота/поилки/кормушки):
- Животное проходит → считывается метка → данные на сервер
- Сервер синхронизируется с SubsidyAI
- Real-time статус: животное живо, на месте, здорово
- Автоматический сигнал в скоринг

### 6.4. Периодичность сканирования
- Ежедневно (при проходе через станцию) → автоматически
- Еженедельный отчёт → агрегация данных
- Ежемесячный → синхронизация с SubsidyAI для скоринга
- Ежеквартальный → совпадает с текущим мониторингом по Приказу

### 6.5. Влияние на скоринг
- Животное регулярно сканируется → +балл к фактору "Эффективность"
- Пропуски сканирования → снижение балла / предупреждение
- Полное отсутствие данных → сигнал для проверки

### 6.6. Целесообразность по видам

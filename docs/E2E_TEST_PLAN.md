### БЛОК 1: ЗАЯВИТЕЛЬ (farmer1)

---

#### S1.1 — Успешная подача заявки (племенной КРС, «голова»)

**Предусловие**: farmer1 залогинен

1. Перейти на `/auth/login/`
2. Ввести логин `farmer1`, пароль `farmer1`, нажать "Войти"
3. **Проверка**: URL = `/dashboard/`, страница содержит "Панель заявителя"
4. Перейти на `/applications/new/`
5. **Шаг 1 (Регион)**: Выбрать любой регион из списка, район подтянется автоматически
6. **Шаг 2 (Заявитель)**: ИИН/БИН и ФИО должны автозаполниться из профиля farmer1
   - **Проверка**: Поле ИИН/БИН не пустое (12 цифр)
   - **Проверка**: Появился блок "Данные из госсистем" (зелёные/красные бейджи)
7. **Шаг 3 (Хозяйство)**: Показаны данные: количество голов, гектары, ЭСФ
   - **Проверка**: Есть текст "голов" и "га"
8. **Шаг 4 (Субсидия)**:
   - Выбрать категорию "Животноводство"
   - Выбрать направление "Мясное скотоводство" (🐄)
   - Выбрать вид субсидии с единицей "голова" (например "Приобретение маточного поголовья КРС — отечественный")
   - **Проверка**: Нет красного предупреждения о дубликате
   - Нажать "Далее: Выбор животных"
9. **Шаг 5 (Животные)**:
   - **Проверка**: Появился список животных с чекбоксами
   - **Проверка**: Животные с "Ранее субсидировано" — disabled, нельзя выбрать
   - **Проверка**: Животные с "Не подходит по возрасту" — disabled
   - **Проверка**: Подходящие животные автоматически выбраны
   - Оставить >=1 животное выбранным
   - Нажать "Далее: Расчёт суммы"
10. **Шаг 6 (Сумма)**:
    - **Проверка**: Количество = число выбранных животных
    - **Проверка**: Ставка > 0 ₸
    - **Проверка**: Итого = Количество × Ставка
    - Поле "Фактическая стоимость покупки" — ввести число больше чем ставка×2 (чтобы прошла проверка 50%)
    - **Проверка**: Нет красного "Подача невозможна"
    - Нажать "Далее"
11. **Шаг 7 (Документы)**: Пропустить (опционально)
12. **Шаг 8 (Подтверждение)**:
    - **Проверка**: Сводка: регион, заявитель, направление, кол-во, ставка, итого
    - Поставить галочку ЭЦП
    - Нажать "Подать заявку"
13. **Проверка**: Редирект на `/applications/<id>/success/`
14. **Проверка**: Текст "Заявка … создана"
15. Перейти по ссылке "Перейти к заявке" → `/applications/<id>/`
16. **Проверка**: Страница деталей содержит:
    - Номер заявки
    - Статус (не пустой)
    - Секция "Скоринг" или "Hard Filters"
    - Итоговый балл (число) ИЛИ причины отказа

### БЛОК 10: ПОЛНЫЙ ЦИКЛ (end-to-end)

---

#### S10.1 — Полный цикл заявки: подача → скоринг → комиссия → выплата

**Это главный сценарий — проверяет весь бизнес-процесс.**

1. **farmer1**: Залогиниться, подать новую заявку (шаги из S1.1)
   - Запомнить `app_id` и `app_number`
2. **farmer1**: Перейти на `/applications/<app_id>/`
   - **Проверка**: Статус = "approved" или "checking" или "rejected"
   - Если "rejected" → записать причины, перейти к S10.2
3. **specialist**: Залогиниться, перейти на `/applications/<app_id>/`
   - **Проверка**: Видна заявка farmer1, скоринг посчитан
   - Если статус "checking" → нажать "Одобрить"
   - **Проверка**: Статус стал "approved"
4. **commission**: Залогиниться, перейти на `/applications/<app_id>/`
   - **Проверка**: Видна заявка
   - Нажать "Одобрить" (голос комиссии)
5. **head**: Залогиниться, перейти на `/applications/<app_id>/`
   - **Проверка**: Видна заявка
   - Нажать "Выплатить"
   - **Проверка**: Статус = "paid"

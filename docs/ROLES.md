# Роли и права доступа

## Роли в системе

### 1. Заявитель (applicant)
**Кто:** Физическое лицо, юридическое лицо, с/х производственный кооператив (СПК)

**Может:**
- Регистрация / вход через ЭЦП
- Подача заявки (выбор направления из 16 форм)
- Заполнение данных: ИИН/БИН, поголовье, порода, возраст, ЭСФ, земля
- Загрузка документов (договор, ЭСФ, таможенная декларация и т.д.)
- Просмотр статуса своих заявок
- Просмотр истории своих субсидий
- Получение уведомлений (одобрение, отказ, запрос доп. документов)
- Просмотр своей позиции в листе ожидания

**Не может:**
- Видеть заявки других заявителей
- Видеть скоринг-баллы
- Принимать решения по заявкам

---

### 2. Специалист МИО (mio_specialist)
**Кто:** Сотрудник управления сельского хозяйства области/района (МИО)

**Может:**
- Просмотр всех заявок своего региона (области)
- Просмотр карточки заявки с данными из 7 систем
- Просмотр результатов автоматических hard filter проверок
- Запуск ручной проверки (если автоматическая не прошла)
- Запрос дополнительных документов от заявителя
- Проверка документов: ЭСФ, договоры, акты, сертификаты
- Формирование мотивированного отказа (Приложение 7)
- Формирование уведомления о результатах (Приложение 6)
- Просмотр аналитики по своему региону

**Не может:**
- Видеть заявки других регионов
- Менять скоринг-баллы
- Одобрять выплаты (только комиссия)
- Настраивать параметры скоринга

---

### 3. Член комиссии (commission_member)
**Кто:** Член комиссии по субсидированию при МИО

**Может:**
- Просмотр ранжированного шортлиста (по баллам, от высшего к низшему)
- Просмотр карточки заявки со scoring breakdown
- Просмотр explainability report — вклад каждого фактора
- Просмотр данных из всех 7 систем
- Одобрение заявки → отправка на выплату
- Отклонение заявки → мотивированный отказ (Приложение 7)
- Отправка на дополнительную проверку
- Частичное одобрение (при нехватке бюджета)
- Просмотр сводной аналитики по всем заявкам

**Не может:**
- Редактировать данные заявок
- Менять скоринг-баллы
- Управлять пользователями

---

### 4. Руководитель МИО (mio_head)
**Кто:** Руководитель управления сельского хозяйства области

**Может:**
- Все что может специалист МИО
- Все что может член комиссии
- Просмотр аналитики по всем районам области
- Просмотр бюджета: план / факт / остаток по направлениям
- Утверждение решений комиссии (финальная подпись)
- Просмотр истории решений комиссии с обоснованиями
- Экспорт отчетов (Excel, PDF)
- Мониторинг KPI: % освоения бюджета, среднее время обработки

**Не может:**
- Менять скоринг-баллы
- Управлять пользователями системы
- Настраивать модель скоринга

---

### 5. Администратор системы (admin)
**Кто:** Технический администратор платформы SubsidyAI

**Может:**
- Управление пользователями: создание, блокировка, роли
- Настройка весов скоринговой модели (soft factors)
- Настройка нормативов субсидий (Приложение 1)
- Управление справочниками: регионы, направления, породы, статусы
- Управление эмулятором: запуск генерации данных, просмотр API
- Просмотр логов системы
- Экспорт любых данных
- Управление бюджетами по регионам и направлениям

**Не может:**
- Принимать решения по заявкам (не член комиссии)

---

### 6. Аудитор (auditor) — только чтение
**Кто:** Контролирующий орган, внутренний аудит

**Может:**
- Просмотр всех заявок всех регионов (read-only)
- Просмотр всех решений комиссии с обоснованиями
- Просмотр scoring breakdown любой заявки
- Просмотр аналитики по всей республике
- Просмотр логов действий пользователей
- Экспорт отчетов

**Не может:**
- Создавать/редактировать заявки
- Принимать решения
- Менять настройки системы

---

## Региональная фильтрация

- **Заявитель** → видит только свои данные
- **Специалист МИО** → только заявки своей области
- **Комиссия** → только заявки своей области
- **Руководитель МИО** → все районы своей области
- **Админ** → все регионы
- **Аудитор** → все регионы (read-only)

## Авторизация в MVP (хакатон)

Для демо: выбор роли через dropdown на странице логина (без реальной ЭЦП). Каждая роль переключает интерфейс и доступные функции.

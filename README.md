# SubsidyAI — AI-скоринг для справедливого распределения сельскохозяйственных субсидий

## Проблема

Текущая система распределения субсидий в Казахстане работает по принципу **FIFO** (кто первый подал заявку — тот и получил). Это приводит к:
- Несправедливому распределению средств
- Коррупционным рискам
- Отсутствию объективной оценки заявителей
- Невозможности приоритизации эффективных хозяйств

## Решение

**SubsidyAI** заменяет FIFO-очередь на **AI-скоринг**: каждая заявка получает балл от 0 до 100 на основе данных из 7 государственных информационных систем. Рейтинг формируется автоматически, а комиссия принимает решения на основе прозрачных данных.

### Ключевые возможности

| Возможность | Описание |
|-------------|----------|
| **Гибридный AI-скоринг** | 60% ML (Gradient Boosting, 200 деревьев) + 40% экспертные правила |
| **20 признаков из 7 систем** | ГИСС, ИАС РСЖ, ЕАСУ, ИС ИСЖ, ИС ЭСФ, ЕГКН, Казначейство |
| **15 жёстких фильтров** | Бинарные проверки по Приказу №108 МСХ РК (pass/fail) |
| **8 мягких факторов** | Взвешенная оценка по 8 параметрам (макс. 100 баллов) |
| **SHAP Explainability** | TreeExplainer — вклад каждого параметра в AI-оценку с формулами |
| **9-шаговая форма подачи** | Farmer-friendly интерфейс с пре-валидацией на каждом шаге |
| **RFID-мониторинг** | Интеграция с RFID-метками для отслеживания поголовья |
| **Спутниковая карта** | Esri World Imagery + рисование границ земельных участков |
| **Личный кабинет фермера** | "Мое хозяйство" + "Моя аналитика" — полная картина данных |
| **Комиссия решает** | AI рекомендует, человек принимает финальное решение |
| **Аудиторский след** | Кто решил, когда, почему — полная прослеживаемость |
| **6 ролей** | Заявитель, специалист, комиссия, руководитель, аудитор, админ |

## Быстрый старт

### Требования
- **Docker** + **Docker Compose** (единственное требование)
- ОС: Linux, macOS, Windows (WSL2)
- Свободно: ~2 ГБ RAM, ~1 ГБ диск

### Запуск (одна команда)

```bash
git clone https://github.com/azamatkajyrov26-lab/hackathon-ai.git
cd hackathon-ai
docker compose up --build -d
```

> Все переменные окружения имеют значения по умолчанию. Файл `.env` не требуется для локального запуска.

Система автоматически:
1. Создаст БД PostgreSQL 16 и Redis 7
2. Применит миграции Django
3. Загрузит справочники (25 направлений, 120 видов субсидий, 17 регионов)
4. Создаст демо-пользователей (13 аккаунтов: 5 фермеров, 2 специалиста, 2 комиссия, 2 руководителя, 1 админ, 1 аудитор)
5. Сгенерирует 500 эмулированных фермерских хозяйств с RFID-данными
6. Создаст 300 тестовых заявок и проведёт AI-скоринг
7. Обучит ML модель (GradientBoosting, 200 деревьев, 20 признаков)
8. Запустит Gunicorn на порту 8000

**Первый запуск: ~2-3 минуты** (обучение модели + скоринг). Далее:

```
http://localhost:8001
```

## Датасет

Файл: **[`data/sample_dataset.json`](data/sample_dataset.json)**

Синтетический датасет из 5 эмулированных хозяйств с полными данными из 7 государственных систем. Используется как reference для понимания структуры данных.

### Что внутри

| Раздел | Описание |
|--------|----------|
| `entities` | 5 хозяйств разного профиля риска (clean, minor_issues, risky, fraudulent) |
| `subsidy_directions` | 25 направлений субсидирования (животноводство, растениеводство, техника, переработка) |
| `subsidy_types_sample` | 6 примеров видов субсидий с нормативами и ограничениями |
| `regions` | 17 областей Казахстана |
| `scoring_config` | Конфигурация AI-скоринга: формула, факторы, пороги |
| `ml_features` | 20 признаков ML модели с описанием и источниками |

### Структура данных одного хозяйства (EmulatedEntity)

```
entity
├── iin_bin            # ИИН/БИН (12 цифр)
├── name               # Наименование (ТОО, КХ, ИП, СПК)
├── entity_type        # individual | legal | cooperative
├── region / district  # Область / Район
├── risk_profile       # clean | minor_issues | risky | fraudulent
│
├── giss_data          # ГИСС — статистика, рост, обязательства, блокировки
├── ias_rszh_data      # ИАС РСЖ — история субсидий, возвраты
├── easu_data          # ЕАСУ — учётные номера, членство в СПК
├── is_iszh_data       # ИС ИСЖ — животные (бирка, порода, возраст, RFID, вет. статус)
├── is_esf_data        # ИС ЭСФ — электронные счёт-фактуры
├── egkn_data          # ЕГКН — земельные участки (кадастр, площадь, назначение)
└── treasury_data      # Казначейство — история платежей
```

### Профили риска

| Профиль | Доля | Характеристики |
|---------|------|----------------|
| **clean** | 70% | ГИСС зарегистрирован, рост продукции 2-20%, обязательства выполнены, нет блокировок |
| **minor_issues** | 15% | Рост -3…+10%, часть обязательств не выполнена |
| **risky** | 10% | Снижение -15…+3%, есть невозвраты, ветеринарный карантин |
| **fraudulent** | 5% | Снижение -25…-5%, блокировки, ранее субсидированные животные, ЭСФ-черновики |

### 8 мягких факторов скоринга (правила)

| # | Фактор | Макс. | Вес | Источник | Формула |
|---|--------|-------|-----|----------|---------|
| 1 | История субсидий | 20 | 0.20 | ИАС РСЖ + БД | Успех ≥90% → 18-20; 70-90% → 14-18; 50-70% → 8-14; <50% → 0-8. Штраф за невыполнение встречных обязательств (Приказ №108) |
| 2 | Рост валовой продукции | 20 | 0.20 | ГИСС | Рост ≥10% → 20; 5-10% → 14-20; 0-5% → 8-14; снижение → 0-8 |
| 3 | Размер хозяйства | 15 | 0.15 | ЕГКН + ИС ИСЖ | Земля (0-8 баллов) + Поголовье (0-7 баллов) |
| 4 | Эффективность | 15 | 0.15 | ИАС РСЖ + ГИСС | Сохранность ≥98% → 15; бонус/штраф за встречные обязательства |
| 5 | Соответствие нормативу | 10 | 0.10 | ИС ЭСФ | Норматив + ЭСФ → 10; частичное → 4-6; несоответствие → 2 |
| 6 | Региональный приоритет | 10 | 0.10 | Справочник МСХ | Приоритет × 10 (Костанайская мясо → 0.9, Туркестанская мясо → 0.5) |
| 7 | Тип заявителя | 5 | 0.05 | ЕАСУ | СПК → 5, Юрлицо → 4, Физлицо → 3 |
| 8 | Первичность заявителя | 5 | 0.05 | ИАС РСЖ + БД | Первичный → 4, Диверсификация → 5, Умеренный → 3, Частый → 2 |

## Ограничения

| Ограничение | Описание |
|-------------|----------|
| **Синтетические данные** | Все данные сгенерированы Faker + random. Не отражают реальных фермеров РК |
| **Эмуляция госсистем** | 7 ИС эмулируются через JSON. В продакшене нужна интеграция через API МСХ |
| **ML модель** | Обучается на синтетических данных → точность отражает качество генерации, не реальные паттерны |
| **Без аутентификации по ЭЦП** | Вход через ЭЦП имитируется вводом ИИН/БИН. Нет интеграции с NCALayer |
| **Один регион** | Бюджеты и рейтинги работают в рамках одного запуска, нет федеративной агрегации |
| **Нет мобильного приложения** | Только веб-интерфейс (адаптивный дизайн для мобильных) |
| **RFID-мониторинг** | Данные сканирования эмулированы, нет интеграции с реальными UHF/LF ридерами |
| **Нет уведомлений** | Push/SMS/email уведомления не реализованы (только in-app) |

## Сценарии применения

### 1. Демонстрация AI-скоринга на хакатоне
```
Войти как farmer1 → Подать заявку → Увидеть балл, формулы, SHAP-объяснения
Войти как specialist → Просмотреть рейтинг → Аналитика
Войти как commission → Принять решение → Инициировать выплату (head)
```

```
Войти через ЭЦП как 750401301234 (Марат Омаров, заблокирован) → Попытка подачи → Отказ
Войти через ЭЦП как 920301512345 (Богдан Сидоров, СПК) → Подача → Высокий балл

```

### 3. Анализ прозрачности AI
```
Открыть любую заявку → Секция "AI-скоринг"
├── Формула: ML_score × 0.6 + rule_score × 0.4 = итого
├── SHAP: вклад каждого из 20 параметров (полоски + объяснения)
└── 8 факторов: нажать на каждый → таблица входных данных + формула расчёта
```

### 4. Интеграция с реальными данными
```
1. Заменить EmulatedEntity на API-клиенты к ГИСС, ИАС РСЖ, ЕАСУ, ИС ИСЖ, ИС ЭСФ, ЕГКН, Казначейству

```

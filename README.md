# SubsidyAI — AI-скоринг для справедливого распределения сельскохозяйственных субсидий

## Проблема

Текущая система распределения субсидий в Казахстане работает по принципу **FIFO** (кто первый подал заявку — тот и получил). Это приводит к:
- Несправедливому распределению средств
- Коррупционным рискам
- Отсутствию объективной оценки заявителей
- Невозможности приоритизации эффективных хозяйств

## Решение

**SubsidyAI** заменяет FIFO-очередь на **AI-скоринг**: каждая заявка получает балл от 0 до 100 на основе данных из 7 государственных информационных систем. Рейтинг формируется автоматически, а комиссия принимает решения на основе прозрачных данных.

### Ключевые возможности

| Возможность | Описание |
|-------------|----------|
| **Гибридный AI-скоринг** | 60% ML (Gradient Boosting, 200 деревьев) + 40% экспертные правила |
| **20 признаков из 7 систем** | ГИСС, ИАС РСЖ, ЕАСУ, ИС ИСЖ, ИС ЭСФ, ЕГКН, Казначейство |
| **15 жёстких фильтров** | Бинарные проверки по Приказу №108 МСХ РК (pass/fail) |
| **8 мягких факторов** | Взвешенная оценка по 8 параметрам (макс. 100 баллов) |
| **9-шаговая форма подачи** | Farmer-friendly интерфейс с пояснениями к каждому полю |
| **RFID-мониторинг** | Интеграция с RFID-меткими для отслеживания поголовья |
| **Спутниковая карта** | Esri World Imagery + рисование границ земельных участков |
| **Личный кабинет фермера** | "Мое хозяйство" + "Моя аналитика" — полная картина данных |
| **Объяснимость (Explainability)** | Текстовое обоснование на русском + вклад каждого фактора |
| **Комиссия решает** | AI рекомендует, человек принимает финальное решение |
| **Аудиторский след** | Кто решил, когда, почему — полная прослеживаемость |
| **6 ролей** | Заявитель, специалист, комиссия, руководитель, аудитор, админ |

## Быстрый старт

### Требования
- **Docker** + **Docker Compose** (единственное требование)

### Запуск (одна команда)

```bash
git clone https://github.com/azamatkajyrov26-lab/hackathon-ai.git
cd hackathon-ai
docker compose up --build -d
```

> Все переменные окружения имеют значения по умолчанию. Файл `.env` не требуется для локального запуска.

Система автоматически:
1. Создаст БД PostgreSQL и Redis
2. Применит миграции
3. Загрузит справочники (25 направлений, 114 видов субсидий, 17 регионов)
4. Создаст демо-пользователей (служебные + 6 тестовых фермеров)
5. Сгенерирует 500 эмулированных фермерских хозяйств с RFID-данными
6. Создаст 300 тестовых заявок
7. Проведёт AI-скоринг всех заявок
8. Обучит ML модель
9. Запустит веб-сервер

**Первый запуск занимает ~2-3 минуты** (обучение модели + скоринг). Далее открывайте:

```
http://localhost:8000
```

### Пайплайн скоринга

```
Заявка подана (9-шаговая форма)
    │
    ▼
┌─────────────────────────────────────────────────┐
│  1. Загрузка данных из 7 госсистем (эмулятор)   │
└────────────────────┬────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────┐
│  2. 15 жёстких фильтров по Приказу №108         │──→ fail = автоматический отказ
│     (3 критических + 4 мягких + 8 стандартных)  │
└────────────────────┬────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────┐
│  3. 8 мягких факторов → rule_score (0-100)      │
└────────────────────┬────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────┐
│  4. ML модель (20 признаков) → ml_score (0-100) │
└────────────────────┬────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────┐
│  5. Гибрид: 0.6 × ML + 0.4 × Rules = итог      │
│                                                   │
│  ≥70 → Одобрить  │ ≥40 → На усмотрение │ <40 → Отклонить │
└────────────────────┬────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────┐
│  6. Проверка бюджета → статус заявки            │
│  7. Объяснение AI на русском                    │
│  8. Ранжирование в рейтинге                    │
│  9. Уведомление заявителю                       │
└─────────────────────────────────────────────────┘
                     ▼
         Комиссия → Решение → Выплата
```

### 7 информационных систем (эмулятор)

| # | Система | Данные | Что проверяется |
|---|---------|--------|-----------------|
| 1 | **ГИСС** | Статистика, рост производства | Регистрация, обязательства, блокировки |
| 2 | **ИАС РСЖ** | История субсидий | Регистрация, ИБСПР, возвраты |
| 3 | **ЕАСУ** | Учётные номера | Наличие ЭЦП |
| 4 | **ИС ИСЖ** | Поголовье, верификация | Идентификация, возраст, субсидированность |
| 5 | **ИС ЭСФ** | Электронные счета-фактуры | Подтверждение затрат |
| 6 | **ЕГКН** | Земельный кадастр | Наличие с/х земли, площадь, границы участков |
| 7 | **Казначейство** | Платежи | Статус выплат |

> Все системы эмулируются через JSON в `EmulatedEntity`. В продакшене заменяются реальными API МСХ РК.

### RFID-мониторинг

Система интегрирована с RFID-технологией для идентификации и мониторинга поголовья:

- **UHF RFID** — для крупного рогатого скота (расстояние считывания до 12м)
- **LF RFID** — для мелкого рогатого скота (болюсные метки)
- Дашборд с графиком активности сканирований за 30 дней
- Расчёт стоимости оборудования (Комплект A / B / C) в зависимости от поголовья
- Таблица животных с фильтрацией по статусу RFID

### Карта земельных участков

- **Esri World Imagery** — спутниковые снимки с зумом до 22 (видны поля, постройки)
- **Leaflet.draw** — рисование полигонов для обводки границ участков
- Сохранение GeoJSON-границ в данных ЕГКН
- Переключатель слоёв: спутник / карта

### 20 признаков ML модели

Gradient Boosting анализирует: регистрация в ГИСС, темп роста производства, валовая продукция (2 года), выполнение обязательств, объём полученных субсидий, регистрация ИАС, количество субсидий, успешность субсидий, возвраты, верифицированные/отклонённые животные, валидность возраста, сумма ЭСФ, количество ЭСФ, доля подтверждённых, наличие земли, площадь, тип субъекта, платежи казначейства.

## Локальная разработка (без Docker)

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# Нужны PostgreSQL и Redis
export DB_NAME=hackathon DB_USER=hackathon DB_PASSWORD=hackathon DB_HOST=localhost
export REDIS_URL=redis://localhost:6379/0

python manage.py migrate
python manage.py seed_data
python manage.py generate_data --count 500
python manage.py create_demo_users
python manage.py create_test_applications --count 300
python manage.py train_model
python manage.py runserver
```
